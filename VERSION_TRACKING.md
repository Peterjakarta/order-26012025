# Automatic Version Tracking

Your application now automatically tracks version history from Git commits. No more manual entries or Firebase permissions issues!

## How It Works

### Automatic on Build
Every time you run `npm run build`, the system automatically:
1. Reads your Git commit history
2. Categorizes commits by type (feature, bugfix, performance, etc.)
3. Generates a version file with all commit details
4. Includes this in your built application

### Viewing Version History
Go to **Settings → Version Info** in your app to see:
- Current version number
- All commits grouped by version
- Each commit categorized and color-coded
- Release dates and commit counts

## Commit Message Conventions

The system automatically categorizes your commits based on message keywords:

| Keyword | Category | Color |
|---------|----------|-------|
| `feat:`, `add`, `implement` | Feature | Blue |
| `fix:`, `bug` | Bugfix | Red |
| `perf:`, `optimize`, `performance` | Performance | Orange |
| `security`, `auth` | Security | Green |
| `enhance`, `improve`, `refactor` | Enhancement | Purple |
| `docs:`, `documentation` | Documentation | Gray |

### Examples

```bash
# These will be categorized as Features
git commit -m "feat: add new product export feature"
git commit -m "add support for batch printing"
git commit -m "implement user notifications"

# These will be categorized as Bugfixes
git commit -m "fix: resolve login error on mobile"
git commit -m "bug: correct price calculation"

# These will be categorized as Performance
git commit -m "perf: optimize database queries"
git commit -m "improve loading speed for reports"

# These will be categorized as Documentation
git commit -m "docs: update README with setup instructions"
git commit -m "add API documentation"
```

## Manual Update

If you want to update version history without building:

```bash
npm run update-version
```

This will read your Git commits and regenerate the version file.

## Version File Location

The generated version data is stored at:
```
src/data/versions.ts
```

This file is automatically generated - **do not edit it manually**!

## Troubleshooting

### "Not a git repository" warning
This means you're not in a Git repository. The system will create a default version file. Once you initialize Git and make commits, it will start tracking automatically.

### No commits showing
Make sure you've committed your changes to Git before building:
```bash
git add .
git commit -m "feat: your changes here"
npm run build
```

### Commits not categorized correctly
Check your commit message uses the keywords listed above. The system looks for these patterns to auto-categorize.
